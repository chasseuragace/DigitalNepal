GET /v1/nid/verify — simplest call, returns only true/false

// Request — app sends NID number, gets back boolean only
GET https://api.nid.gov.np/v1/nid/verify
Authorization: Bearer <oauth2_access_token>
X-Consent-Token: <citizen_issued_consent_uuid>

{
  "nid": "1234-5678-9012",
  "purpose": "account_opening",
  "app_id": "nabil_bank_kyc_prod"
}

// Response — nothing more than what was asked
{
  "valid": true,
  "query_id": "q_8f3a2b1c",    // for audit trail
  "timestamp": "2025-03-22T09:14:22Z",
  "consent_expiry": "2025-03-22T09:44:22Z"
}

// Error: no consent token issued
{
  "error": "consent_required",
  "consent_url": "https://consent.nid.gov.np/approve?app=nabil_bank&scope=verify"
}

Citizen consent flow (what happens before this API call works)

// Step 1: App redirects citizen to consent portal
https://consent.nid.gov.np/approve
  ?app_id=nabil_bank_kyc_prod
  &scope=verify
  &redirect_uri=https://nabilbank.com/kyc/callback
  &state=csrf_token_here

// Step 2: Citizen logs in with NID + OTP, approves scope
// Step 3: NID portal redirects back with consent token
https://nabilbank.com/kyc/callback
  ?consent_token=ct_9f2e8d3a1b
  &state=csrf_token_here
  &expires_in=1800    // 30 minutes

Authorization: Bearer <oauth2_access_token>

Critical risks — what kills each initiative

HIGH NID DB data quality. ~40% of NID records in Nepal have inconsistent romanization of names, missing ward numbers, or mismatched DOB vs. certificate data. Any API that returns bad data destroys trust fast. Mitigation: Phase 0 audit is not optional.
HIGH NRB political resistance. NRB has historically been conservative on FX liberalization. The sandbox circular needs explicit Cabinet endorsement, not just NRB governor's approval. Without RSP's parliamentary majority actively pushing, this stalls. Timeline impact: +6–12 months.
HIGH Procurement capture. Both systems are ~NPR 80–130M contracts — attractive for inflated tendering. If awarded to politically connected IT firms without technical capacity, both systems ship broken. Mitigation: open-source stack, public code repository, technical evaluation committee with private sector members.
MED Privacy legislation gap. Nepal has no comprehensive data protection law. Without it, the consent framework in the NID API has no legal teeth — companies can query without genuine consent and face no penalty. A data protection ordinance should precede Phase 2 of NID API.
MED Single point of failure. If the NID API goes down, every KYC-dependent service in Nepal stops. Uptime SLA must be 99.9%+ with hot standby. On Nepal's current NTC infrastructure this requires redundant DC setup — not just redundant servers.
LOW Developer adoption. Nepal has a growing but small dev community (~20K engineers). A sandbox with good docs and active Discord/community support should attract 200–500 pilot integrations in year 1. India's Aadhaar got 1000 integrations in 18 months from a much larger base.
